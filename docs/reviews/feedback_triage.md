# encodeUtils feedback triage

Date: 2026-07-03

Superseded note: this is a historical triage snapshot. Use
`docs/reviews/consolidated_audit_synthesis.md` as the current audit and
implementation reference.

This note records how the external review feedback was handled during the source
audit.

## Accepted and addressed

- Default search/object richness: `encode_search()` and `encode_get()` now
  default to `frame = "embedded"` because live ENCODE checks showed the previous
  object-frame default returned lab slugs, award IDs, and missing biosample
  classification in compact tables.
- ENCODE preferred defaults: file metadata now includes `preferred_default`,
  `analyses`, and `analysis_step_version`; `encode_select_files()` has a
  `prefer_default` path that is used only when preferred-default rows are
  actually present.
- Selection presets: `encode_select_files()` now uses the clearer `preset`
  argument and includes assay-aware presets such as `chipseq_peaks`,
  `chipseq_signal`, `atacseq_peaks`, `rnaseq_gene_quant`, and
  `rnaseq_transcript_quant`.
- Selection reporting: print methods now summarize exclusion reasons so users
  can see why rows were removed.
- Matrix result duplication: `encode_matrix()` now keeps `matrix`,
  `assay_summary`, and `biosample_summary` only.
- Pre-release alias cleanup: removed exported aliases `encode_files()`,
  `encode_schema()`, `encode_citation()`, and `encode_interactive_search()`.
- Download robustness: file verification failures are now returned as per-file
  `download_status = "failed"` rows with `failure_reason`, preserving successful
  rows in the same batch.
- Retry pacing: transient HTTP retries now use `Retry-After` when available and
  exponential backoff otherwise.
- Citation styles: `encode_cite(style = ...)` now produces distinct summary,
  methods, and supplement outputs. File-table citations can optionally enrich
  attribution from parent experiments with `enrich = TRUE`.
- Documentation organization: review and planning notes were moved under
  `docs/`, while package-facing docs remain in README, NEWS, vignettes, and man
  pages.

## Rejected or downgraded

- Missing `encode_select_files.Rd` and `encode_read.Rd`: rejected as stale
  feedback. Both help pages exist in the working tree and are regenerated by
  roxygen.
- Drop `encode_summary()`: rejected. Source review confirmed it is a real
  polymorphic helper, not a trivial alias.
- Add full analysis/default-analysis pinning now: downgraded. The package now
  surfaces `analyses` and `preferred_default`, but a full `analysis = "default"`
  API needs more ENCODE analysis-object modeling and should wait until the file
  selection layer has real usage.
- Add consensus peak generation, SQLite mirroring, Shiny, private auth, or
  automatic FASTQ/BAM processing: rejected as over-scoped or unsafe for the
  package's read-only, metadata-first v1.
